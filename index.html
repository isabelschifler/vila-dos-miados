📐 Função Relevante

Função de Esforço Felino: E(u,v) = d + 2r + s

Onde:

  • d = distância: 1 (pulo curto), 2 (médio), 3 (longo), 4 (muito longo)
  • r = risco: 0 (seguro), 1 (escuro), 2 (obstáculos), 3 (cães/perigo alto)
  • s = salto: 0 (mesma altura/fácil), 1 (pulo pequeno), 2 (pulo arriscado)

🧮 3. Algoritmo de Dijkstra - Passo a Passo

Teste 1: Telhado (T) → Janela (J)

Passo 1: Distâncias iniciais: T=0, B=∞, P=∞, M=∞, J=∞
Visitados: {T}
Passo 2: Visitando T → relaxando arestas:
T→B: 0+3=3 → B=3 ✓
T→P: 0+5=5 → P=5 ✓
T→J: 0+8=8 → J=8 ✓
Distâncias: T=0, B=3, P=5, M=∞, J=8
Passo 3: Menor distância não visitada: B(3). Visitar B.
B→M: 3+2=5 → M=5 (anterior ∞)
B→J: 3+6=9 → J permanece 8 (9 é maior)
Distâncias: T=0, B=3, P=5, M=5, J=8
Passo 4: Menor não visitada: P(5) ou M(5). Visitar P.
P→M: 5+4=9 → M permanece 5 (9 é maior)
Distâncias: T=0, B=3, P=5, M=5, J=8
Passo 5: Visitar M(5).
M→J: 5+1=6 → J atualiza: 8 → 6 ✓
Distâncias finais: T=0, B=3, P=5, M=5, J=6
Passo 6: Visitar J(6). Algoritmo concluído.

✅ Resultado Teste 1:

Caminho mais curto: T → B → M → J

Custo total: 3 + 2 + 1 = 6

(Alternativa direta T→J custa 8, mais caro)

Teste 2: Praça (P) → Janela (J)

Passo 1: Distâncias: P=0, T=∞, B=∞, M=∞, J=∞
Passo 2: Visitar P → P=0, atualiza: T=5, M=4
Distâncias: P=0, T=5, B=∞, M=4, J=∞
Passo 3: Menor não visitado: M(4). Visitar M → M=4, atualiza: J=5 (4+1), B=6 (4+2)
Distâncias: P=0, T=5, B=6, M=4, J=5
Passo 4: Menor não visitado: T(5) ou J(5). Visitar T → T=5, não melhora J (5+8=13)
Distâncias: P=0, T=5, B=6, M=4, J=5
Passo 5: Visitar J(5) → algoritmo concluído.

✅ Resultado Teste 2:

Caminho mais curto: P → M → J

Custo total: 4 + 1 = 5

📊 4. Análise dos Resultados

O algoritmo de Dijkstra encontrou caminhos que não são os mais óbvios:

  • De Telhado à Janela, o caminho direto parecia rápido, mas o algoritmo mostrou que o caminho T→B→M→J é melhor (custo 6 vs 8).
  • De Praça à Janela, o caminho P→M→J foi o escolhido (custo 5), ignorando caminhos mais longos.
  • Isso mostra como o algoritmo avalia combinações de arestas em vez de apenas rotas diretas.

💻 5. Pseudocódigo do Algoritmo

function Dijkstra(grafo, origem, destino):
    distancias[origem] = 0
    for each vértice v in grafo:
        if v != origem:
            distancias[v] = infinito
        nao_visitados.add(v)
    
    while nao_visitados não está vazio:
        u = vértice em nao_visitados com menor distância
        nao_visitados.remove(u)
        
        for each vizinho v de u:
            if v está em nao_visitados:
                nova_dist = distancias[u] + peso(u,v)
                if nova_dist < distancias[v]:
                    distancias[v] = nova_dist
                    predecessor[v] = u
    
    return distancias[destino], reconstruir_caminho(predecessor, destino)
